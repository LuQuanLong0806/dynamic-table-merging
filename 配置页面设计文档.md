# 动态表格配置页面 — 设计文档

> 本文档面向产品和前端开发，说明管理端"动态表格配置"页面的设计方案。
> 配置页面的目标是让管理员通过可视化界面生成 JSON 数据配置，无需手写 JSON。

---

## 一、整体设计思路

### 1.1 核心流程

```
管理员进入配置页面
  → 填写表格基本信息（标题、列数、行数）
  → 在可视化表格中逐个配置每个单元格
  → 配置单元格类型、内容、校验规则
  → 配置控件间的联动关系
  → 预览效果
  → 保存生成 JSON
```

### 1.2 页面布局

```
┌────────────────────────────────────────────────────────────┐
│  动态表格配置中心                              [保存] [预览]  │
├─────────────────────┬──────────────────────────────────────┤
│                     │                                      │
│   左侧：配置面板     │   右侧：实时预览区                     │
│   ─────────────     │   ─────────────                      │
│   1. 基本设置        │   表格实时渲染效果                      │
│   2. 单元格属性      │   所见即所得                           │
│   3. 选项配置        │                                      │
│   4. 校验规则        │                                      │
│   5. 联动配置        │                                      │
│   6. 公式配置        │                                      │
│                     │                                      │
├─────────────────────┴──────────────────────────────────────┤
│  底部：JSON 源码编辑器（可直接编辑 JSON）                       │
└────────────────────────────────────────────────────────────┘
```

## 二、配置页面模块详细设计

### 2.1 基本设置面板

进入配置页面后的第一步。

| 配置项 | 控件类型 | 说明 |
|--------|----------|------|
| 表格标题 | Input | 对应 `title` 字段 |
| 总列数 | InputNumber | 确定表格的最大列数，影响表头行 |
| 总行数 | InputNumber | 确定表格的最大行数（含表头） |
| 列宽设置 | 动态列表 | 每列一个输入框，设置宽度百分比 |

交互：
1. 填写列数和行数后点击"生成表格"
2. 自动生成一个 `行数 × 列数` 的空白表格框架
3. 第1行默认为表头行（type=label），其余行默认为普通行

---

### 2.2 可视化表格编辑区

右侧预览区即是编辑区。管理员点击任意单元格，左侧面板显示该单元格的配置选项。

#### 2.2.1 单元格选择交互

- **单击**：选中单元格，左侧面板显示属性配置
- **拖拽选区**：选中多个连续单元格，弹出"合并"按钮
- **右键菜单**：合并单元格 / 取消合并 / 插入行 / 删除行

#### 2.2.2 合并操作

```
┌───┬───┬───┐       ┌───┬───────┐
│ A │ B │ C │       │ A │合并区域│
├───┼───┼───┤  →──→ ├───┼───┬───┤
│ D │ E │ F │       │ D │ E │ F │
└───┴───┴───┘       └───┴───┴───┘

选中 B+C → 点击"横向合并" → x:"2,3", colspan:2
```

- **横向合并**：选中的列合并为一个单元格，自动计算 `x: "startCol,endCol"`
- **纵向合并**：选中的行合并为一个单元格，自动计算 `y: "startRow,endRow"`
- **取消合并**：恢复为独立单元格

> 配置页面自动计算 `x`、`y` 坐标和合并范围，管理员无需关心坐标值。

---

### 2.3 单元格属性配置面板

点击单元格后，左侧面板显示：

#### 基本属性

| 配置项 | 控件类型 | 说明 |
|--------|----------|------|
| 坐标信息 | 只读文本 | 显示 `x:3, y:2`（自动计算） |
| 单元格类型 | Select | `label` / `text` / `input` / `number` / `select` / `radio` / `checkbox` / `date` / `formula` |
| 显示内容 | Input (textarea) | 对应 `content`，支持 HTML |
| 初始值 | Input | 对应 `value` |
| 占位提示 | Input | 对应 `placeholder` |

#### 类型切换动态面板

选择不同 `type` 后，动态显示不同的配置项：

**type = label / text**：
- 只需配置 `content`（显示内容）

**type = input**：
- content（显示内容/placeholder）
- value（初始值）
- validation（校验规则，见下文）

**type = number**：
- content / value / placeholder
- validation
- disabled（初始禁用）

**type = select**：
- content（下拉框 placeholder 提示）
- value（初始值）
- options（选项配置，见下文）
- validation
- linkage（联动配置，见下文）

**type = radio**：
- content（提示文字）
- value
- options（选项配置）
- validation
- linkage

**type = checkbox**：
- options
- value（数组初始值）
- validation

**type = date**：
- value
- validation

**type = formula**：
- calculationFormula（公式编辑器，见下文）

### 2.4 选项配置（options）

当 type 为 `select`/`radio`/`checkbox` 时显示。

```
┌──────────────────────────────────────────┐
│  选项列表                    [+ 添加选项]  │
├────────────┬────────────┬────────────────┤
│  显示文字   │    值      │    操作         │
├────────────┼────────────┼────────────────┤
│  有适用标准  │    has    │  [↑][↓][删除]   │
│  无适用标准  │    no     │  [↑][↓][删除]   │
└────────────┴────────────┴────────────────┘
```

| 操作 | 说明 |
|------|------|
| 添加选项 | 新增一行空的 label + value |
| 上移/下移 | 调整选项显示顺序 |
| 删除 | 移除该选项 |

配置完成后自动生成：
```json
"options": [
  {"label":"有适用标准","value":"has"},
  {"label":"无适用标准","value":"no"}
]
```

### 2.5 校验规则配置（validation）

当 type 为可交互控件时显示。

```
┌──────────────────────────────────────────────────────┐
│  校验规则                               [+ 添加规则]   │
├──────────────────────────────────────────────────────┤
│  规则 1                                               │
│  ┌─────────────┬─────────────┬──────────────────┐    │
│  │ 类型: 必填 ▾  │ 触发: 自动 ▾  │ 提示: 请输入标杆值  │    │
│  └─────────────┴─────────────┴──────────────────┘    │
│                                                       │
│  规则 2                                               │
│  ┌─────────────┬────────────────────────────────┐    │
│  │ 类型: 范围 ▾  │ 最小: [0]  最大: [100]           │    │
│  │              │ 提示: 请输入0-100之间的值          │    │
│  └─────────────┴────────────────────────────────┘    │
└──────────────────────────────────────────────────────┘
```

#### 规则类型选择

| 规则类型 | 展开的配置项 | 生成的 JSON |
|----------|-------------|-------------|
| 必填 | message（提示文字） | `{required:true, message:"..."}` |
| 数值范围 | min, max, message | `{type:"number", min:0, max:100, message:"..."}` |
| 长度范围 | min, max, message | `{type:"string", min:0, max:50, message:"..."}` |
| 正则 | pattern, message | `{pattern:"^\\d+$", message:"..."}` |
| 枚举 | enum列表, message | `{type:"enum", enum:["A","B"], message:"..."}` |
| 邮箱 | message | `{type:"email", message:"..."}` |
| 手机号 | message | `{pattern:"^1[3-9]\\d{9}$", message:"..."}` |

> `trigger` 字段无需管理员配置，引擎自动推断。

### 2.6 联动配置（linkage）

当 type 为 `select`/`radio` 时显示。

```
┌──────────────────────────────────────────────────────────────┐
│  联动规则                                       [+ 添加规则]  │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  当选择 [ 有适用标准 ▾ ] 时：    ← trigger 选择（来自 options） │
│  ┌──────────────────────────────────────────────────────┐    │
│  │  动作列表                                [+ 添加动作]  │    │
│  │                                                       │    │
│  │  ① 对 [ c8_标杆值输入框 ▾ ] 执行 [ 启用 ▾ ]           │    │
│  │  ② 对 [ c9_基准值输入框 ▾ ] 执行 [ 启用 ▾ ]           │    │
│  │  ③ 对 [ d_energy ▾ ] 执行 [ 显示 ▾ ]                 │    │
│  │  ④ 对 [ d_none ▾ ] 执行 [ 隐藏 ▾ ]                   │    │
│  └──────────────────────────────────────────────────────┘    │
│                                                               │
│  当选择 [ 无适用标准 ▾ ] 时：                                  │
│  ┌──────────────────────────────────────────────────────┐    │
│  │  ① 对 [ c8_标杆值输入框 ▾ ] 执行 [ 禁用 ▾ ]           │    │
│  │  ② 对 [ c8_标杆值输入框 ▾ ] 执行 [ 清空 ▾ ]           │    │
│  │  ③ 对 [ d_energy ▾ ] 执行 [ 隐藏 + 清空 ▾ ]           │    │
│  └──────────────────────────────────────────────────────┘    │
│                                                               │
│  [+ 添加触发值]                                               │
└──────────────────────────────────────────────────────────────┘
```

#### 动作配置项

| 配置项 | 说明 |
|--------|------|
| 触发值 (trigger) | 下拉选择当前控件的 options value |
| 目标控件 (target) | 下拉选择表格中其他控件的 id（显示：id + 类型 + 坐标） |
| 动作类型 | 下拉选择：`启用` / `禁用` / `显示` / `隐藏` / `清空` / `设置值` |

#### 行绑定动作（批量操作）

```
┌──────────────────────────────────────────────────────────────┐
│  高级：行绑定动作                                 [+ 添加行]   │
├──────────────────────────────────────────────────────────────┤
│  对以下控件批量执行 [ 启用 ▾ ]：                                │
│  ☑ r2_benchmark   ☑ r2_baseline   ☑ r2_weight   ☑ r2_indvalue│
│  （勾选表格中同一行的多个控件）                                   │
└──────────────────────────────────────────────────────────────┘
```

管理员从表格中勾选多个控件，生成 `rowTargets` 结构。

### 2.7 公式配置

当 type = `formula` 时显示。

```
┌──────────────────────────────────────────────────────┐
│  公式编辑器                                           │
│  ┌──────────────────────────────────────────────────┐│
│  │ SUM( [c6] , [e5] , [f5] )                       ││
│  └──────────────────────────────────────────────────┘│
│                                                      │
│  快捷函数：                                           │
│  [SUM] [AVG] [MIN] [CALC_SCORE] [CALC]              │
│                                                      │
│  可引用的单元格：                                      │
│  c6(分值:11)  e5(分值:5)  f5(分值:4)  c8(标杆值)      │
│  点击可插入到公式中                                    │
│                                                      │
│  CALC 常用模板：                                      │
│  [四则运算] [Math.max] [Math.round] [三元表达式]      │
└──────────────────────────────────────────────────────┘
```

交互：
- 点击快捷函数按钮插入函数模板
- 点击 CALC 按钮后插入 `CALC()` 模板，光标定位在括号内
- 点击 CALC 常用模板可快速插入预设表达式：
  - 四则运算：`CALC((c8 - c9) / c10 * 100)`
  - Math.max：`CALC(Math.max(0, c8 - c9))`
  - Math.round：`CALC(Math.round(c8 * 100) / 100)`
  - 三元表达式：`CALC(c8 > c9 ? c8 : c9)`
- 点击"可引用的单元格"列表中的项插入 cell id
- 支持 `CALC_SCORE` 一键全量求和
- 实时显示公式计算结果预览

#### CALC(表达式) 说明

| 特性 | 示例 |
|------|------|
| 四则运算 | `CALC((c8 - c9) / c10 * 100)` |
| Math 函数 | `CALC(Math.max(0, c8 - c9))` |
| 三元表达式 | `CALC(c8 > c9 ? c8 : c9)` |
| 四舍五入 | `CALC(Math.round(c8 * 100) / 100)` |
| 范围裁剪 | `CALC(c8 <= 0 ? 0 : c8 >= 100 ? 100 : c8)` |

规则：
- 表达式内部是标准 JavaScript 语法
- cell id 会被自动替换为对应数值（text 类型取 content 的数值，其他类型取 fm[id]）
- Math 对象的所有方法可用（Math.max, Math.min, Math.round, Math.floor, Math.ceil, Math.abs, Math.pow 等）
- 表达式中不能嵌套 SUM/AVG 等引擎函数（它们不是 JS 原生函数），只能用标准 JS 语法
- 计算出错时显示 "ERR"

## 三、辅助功能

### 3.1 实时预览

右侧预览区实时展示配置效果：
- 表格布局、合并效果
- 控件样式
- 联动交互（可实际操作测试）
- 校验提示

### 3.2 JSON 源码编辑

底部可切换到 JSON 源码编辑模式：
- 左侧可视化编辑 ↔ 右侧 JSON 编辑 双向同步
- 支持直接粘贴 JSON 导入配置
- JSON 格式校验

### 3.3 模板功能

- **保存为模板**：将当前配置保存为可复用模板
- **从模板创建**：基于已有模板快速生成新表格
- **导入 JSON**：上传 JSON 文件直接生成配置

### 3.4 导出

- **导出 JSON**：下载完整配置 JSON 文件
- **导出为后端接口格式**：生成可直接存入数据库的 JSON

## 四、技术实现建议

### 4.1 前端技术方案

| 模块 | 建议方案 |
|------|----------|
| 框架 | Vue 2 或 Vue 3 |
| UI 组件库 | Element UI / Ant Design Vue |
| 表格编辑 | 基于 HTML `<table>` + contenteditable 或使用 Handsontable |
| JSON 编辑器 | Monaco Editor（VS Code 内核）或 CodeMirror |
| 拖拽选区 | 自定义实现或使用 vue-draggable |
| 状态管理 | Vuex / Pinia 管理表格配置状态 |

### 4.2 核心数据模型

配置页面的核心是一个响应式的表格状态对象：

```js
{
  title: "",
  columns: 13,              // 总列数
  rows: 8,                   // 总行数
  colWidths: ["6%","3%",...], // 列宽配置
  cells: [
    {
      id: "auto_generated",   // 自动生成或手动输入
      x: "1",                 // 自动根据位置计算
      y: "1",
      type: "label",
      content: "",
      value: null,
      placeholder: "",
      options: [],
      validation: null,
      linkage: [],
      calculationFormula: null,
      disabled: false,
      _hidden: false
    }
  ]
}
```

### 4.3 关键交互实现

#### 合并操作

```
用户选中多个连续单元格
  → 计算选区范围 (startCol, endCol, startRow, endRow)
  → 弹出"合并"按钮
  → 点击合并
    → 将选区内的多个 cell 合并为一个
    → 更新主 cell 的 x: "startCol,endCol", y: "startRow,endRow"
    → 删除被合并的 cell
    → 标记被合并的位置为 occupied
```

#### 联动配置

```
用户点击某个 select/radio cell
  → 左侧面板显示"联动配置"区域
  → 自动列出该 cell 的所有 option value 作为 trigger 候选
  → 用户选择一个 trigger
  → 自动列出表格中所有可交互控件作为 target 候选
  → 用户选择 target + action type
  → 支持添加多条 action
```

### 4.4 自动 ID 生成策略

| 场景 | 生成规则 | 示例 |
|------|----------|------|
| 表头 | `h` + 列号 | `h1`, `h2`, `h3` |
| 普通文本 | `r` + 行号 + `c` + 列号 | `r2c4`, `r3c8` |
| 含语义 | 手动输入 | `c8_benchmark`, `sec_ind` |

建议提供"自动生成"和"手动输入"两种模式。

## 五、配置页面完整操作流程示例

以配置"能源低碳化"区域的 select 联动为例：

### 步骤 1：创建表格框架
1. 输入标题"通则指标评分表"
2. 列数=13，行数=8
3. 点击"生成表格"

### 步骤 2：配置表头
1. 点击第1行第1列 → type=label → content="一级指标"
2. 依次配置其余表头...

### 步骤 3：配置合并
1. 选中第1列第4~7行 → 右键"纵向合并"
2. 自动生成 `x:"1", y:"4,7"` → content="能源低碳化"

### 步骤 4：配置 select 控件
1. 点击第4行第4列 → type=select
2. content="是否有适用国家强制性标准"
3. 添加选项：{label:"有适用标准", value:"has"}, {label:"无适用标准", value:"no"}

### 步骤 5：配置联动
1. 在联动面板添加规则
2. trigger="has" → actions: enable c8, enable c9, show d_energy, hide d_none
3. trigger="no" → actions: disable c8, clear c8, disable c9, clear c9, hide d_energy, show d_none

### 步骤 6：配置校验
1. 点击 c8 → 添加校验规则
2. 规则1：必填，提示"请输入标杆值"
3. 规则2：数值范围 min=0，提示"标杆值不能为负"

### 步骤 7：预览与保存
1. 切换到预览模式，实际操作验证联动效果
2. 点击"保存"，生成 JSON 并存入后端
