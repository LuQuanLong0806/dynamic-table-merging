# 动态表格配置页面 — 设计文档

> 本文档面向产品和前端开发，说明管理端"动态表格配置器"（[配置页面.html](配置页面.html)）的设计方案。
> 配置页面让管理员通过可视化界面生成 JSON 数据配置（`{title, freezeRows, cells:[...]}`），无需手写 JSON。
> 输出的 JSON 直接喂给生产组件 [dynamic-eval-table.vue](F:/0_workspace/3-省厅扶持库/project-support-library-platform/src/components/dynamic-eval-table/dynamic-eval-table.vue) 渲染。

---

## 一、整体设计思路

### 1.1 核心流程

```
管理员打开配置页面
  → 顶部输入标题、表头固定行数
  → 添加单元格（从模板库选 / 手动配置）
  → 在编辑抽屉里配 cell 字段（id/坐标/类型/校验/联动/事件/状态）
  → 主区实时预览（可切换 portal/platform 视角）
  → 校验预览提交是否通过
  → 导出 / 复制 / 下载 JSON
```

### 1.2 页面布局（单列纵向）

```
┌──────────────────────────────────────────────────────────────────────┐
│  ◆ 动态表格配置器    [标题]  表头固定[N]行                             │
│   [+ 添加单元格] [使用说明] [导入 JSON] [导出 JSON] [复制 JSON] [清空] │
│   [插入列] [插入行] [追加列] [追加行] [删除列] [删除行] [列可见性]      │
│   [↶ 撤销] [↷ 重做]                                                   │
├──────────────────────────────────────────────────────────────────────┤
│   自动预览 ◯  视角 [全部/门户端/管理端]  ◦单元格 ◦重复 ◦缺ID  [刷新][下载]│
│   ┌────────────────────────────────────────────────────────────┐    │
│   │           实时预览区（preview-renderer 子组件）              │    │
│   │  - 表头吸顶（前 N 行冻结）                                    │    │
│   │  - 行号列（axis-row-head）                                    │    │
│   │  - 点 cell 内的彩色徽章 → 打开编辑抽屉                        │    │
│   │  - 点空位 → 在该位置加 cell                                   │    │
│   └────────────────────────────────────────────────────────────┘    │
├──────────────────────────────────────────────────────────────────────┤
│  编辑抽屉（drawer，从右侧滑出，宽 520）：                              │
│   - ID / 类型 / 坐标 / 初始值                                       │
│   - 显示内容 / 占位提示 / 数字精度 / 溢出隐藏 / 列宽权重              │
│   - 选项（select/radio/checkbox）                                    │
│   - 公式（formula）                                                  │
│   - 上传配置（accept / maxCount）                                    │
│   - 必填设置（非必填 / 始终必填 / 条件必填）                          │
│   - 其他校验规则（数值范围 / 数组长度 / 字符串长度 / 正则 / 定长 /     │
│                  枚举 / 自定义表达式 / 函数体）                       │
│   - 联动 linkage（trigger → actions[]，含 rowTargets 行级批量）       │
│   - 事件 events（onChange / onBlur / onInput）                       │
│   - 状态（正常 / 只读 / 门户隐藏 / 管理端隐藏 / 双端隐藏）            │
└──────────────────────────────────────────────────────────────────────┘
```

> 配置页面**没有左侧"双向同步表格编辑区"**。直接通过预览区的彩色徽章点击编辑，避免双向同步的复杂度。

## 二、顶部工具栏

### 2.1 表格基础信息

| 配置项 | 控件 | 字段 | 说明 |
|---|---|---|---|
| 标题 | i-Input | `title` | 表格标题，渲染为 `<h2>` |
| 表头固定行数 | InputNumber(0~maxY) | `freezeRows` | 0 = 仅 axis-row-top 吸顶；>0 = 前 N 行数据行 + axis-row-top 一起吸顶 |

### 2.2 全局动作按钮

| 按钮 | 行为 |
|---|---|
| + 添加单元格 | 打开模板库 modal，选模板后自动定位到 (x:1, y:maxY+1) |
| 📖 使用说明 | 打开帮助 modal（详细解释字段语义、函数签名、公式） |
| 导入 JSON | 选 / 粘贴 JSON → schema 校验 → 应用 |
| 导出 JSON | 打开导出弹窗（含复制 / 下载 / 压缩开关） |
| 复制 JSON | 直接序列化结果到剪贴板 |
| 清空 | 二次确认后清空所有 cell |

### 2.3 行列操作按钮（全部支持 Ctrl+Z 撤销）

| 按钮 | 行为 | 坐标平移规则 |
|---|---|---|
| ⟳ 插入列 | 在第 N 列前插入 M 列 | 插入点后的 cell 整体右移；跨越插入点的合并 cell 范围扩展（colspan+M） |
| ⟳ 插入行 | 在第 N 行前插入 M 行 | 同上（行方向）；联动规则的 rowTargets.row 自动同步 |
| ➕ 追加列 | 末尾追加 M 列 | 等同"在 maxX+1 前插入" |
| ➕ 追加行 | 末尾追加 M 行 | 等同"在 maxY+1 前插入" |
| ✕ 删除列 | 删除第 N 列 | 独占该列的 cell 删除；跨越的合并 cell 范围缩 1；之后 cell 左移 |
| ✕ 删除行 | 删除第 N 行 | 同上；联动 rowTargets 指向被删行的清理或前移 |
| 👁 列可见性 | 批量设某列为 portalHide/platformHide/allHide | 按起始列归类（跨列合并归到起始列） |

### 2.4 撤销 / 重做栈

- `undoStack` / `redoStack` 存配置快照
- 行列操作、cell 增删、字段编辑都入栈
- 撤销快捷键 Ctrl+Z，重做 Ctrl+Shift+Z（或按钮）

## 四、编辑抽屉（cell-card 组件）

抽屉宽 520px，从右侧滑出，内部按字段分组。每个分组按 cell 类型动态显示。

### 4.1 基础字段

| 字段 | 控件 | 适用类型 | 说明 |
|---|---|---|---|
| ID | i-Input | 全部 | 全局唯一，作 fm 的 key、联动 target、公式引用。重复会标红 |
| 类型 type | i-Select | 全部 | 11 种枚举（label/text/input/textarea/number/select/radio/checkbox/date/upload/formula） |
| 列坐标 x | i-Input | 全部 | 单列 `1` 或合并 `3,5` |
| 行坐标 y | i-Input | 全部 | 单行 `2` 或合并 `2,7` |
| 初始值 value | i-Input | 可交互控件 | 初始值 |
| 显示内容 content | i-Input(textarea) | label/text/select/radio | 显示文本，支持 HTML 和 \n |
| 占位提示 placeholder | i-Input | input/number | 占位文字 |
| 数字精度 precision | i-Select | number | 不限/仅整数/小数 1-4 位 |
| 溢出隐藏 clampLines | InputNumber | label/text | 0=全部显示，N=截断到 N 行（hover 看完整） |
| 列宽权重 width | InputNumber | 表头 label（不限 y/span） | 表头第 1/2 行单列 cell 可配；多行表头第一行合并时配第二行 |

### 4.2 选项 options（select / radio / checkbox）

```
┌─────────────────────────────────────────────┐
│ 选项 options                       [+ 添加]  │
│ ┌───────────────────────────────────────┐  │
│ │ 显示文字 | 值 value | 分值 score(选填) | ×│  │
│ │ 显示文字 | 值 value | 分值 score(选填) | ×│  │
│ └───────────────────────────────────────┘  │
└─────────────────────────────────────────────┘
```

每个选项 3 字段：`label`（显示）/ `value`（值）/ `score`（选填，评分场景用）。

### 4.3 公式 calculationFormula（formula 专用）

i-Input textarea，配快捷按钮：
- `SUM(c6,e5,f5)` 求和（text 类型取 content 数字；保留 2 位小数）
- `AVG(c6,e5)` 平均
- `MIN(c6,e5)` 最小值
- `CALC_SCORE` 所有 number cell 求和
- `CALC(Math.max(0, bm-bl))` 标准 JS 表达式（cell id 直接引用，可用 Math.* 全部方法、三元表达式、四舍五入等）

> CALC 表达式内**不能**嵌套 SUM/AVG 等引擎函数（非 JS 原生）。任一依赖值缺失 → 返回 `'ERR'`。

### 4.4 上传配置（upload 专用）

| 字段 | 控件 | 说明 |
|---|---|---|
| 允许格式 accept | i-Input | 后缀逗号分隔，如 `.pdf,.jpg,.png` |
| 最多文件数 maxCount | InputNumber(min=0) | 0 或留空 = 不限；>0 = 最多 N 个 |

> 单文件大小上限由生产端统一控制（默认 50MB），不在配置页设。
> "至少上传 N 个" 在**校验规则**里配 `array + min`，不在 maxCount。
> 每次只能选 1 个文件，可多次上传累积（与生产 yeyooUpload `:multiple="false"` 对齐）。

### 4.5 必填设置 required（可交互控件）

radio 三选一：
- **非必填** (none)
- **始终必填** (always)：`{required:true, message, trigger}`
- **条件必填** (expr)：`{required: <表达式>, message, trigger}` —— 表达式为 truthy 时必填

| 字段 | 控件 | 说明 |
|---|---|---|
| 错误提示 message | i-Input | 如"此项为必填" |
| 触发时机 trigger | i-Select(blur/change) | number/select/radio/upload 默认 change；input/date 默认 blur |

> checkbox 必填 = 至少选 1 项（生产自动按 array 校验）。
> upload 必填 = 至少传 1 个文件。

### 4.6 其他校验规则（可交互控件）

每条规则一个 `kind`：

| kind | 字段 | 适用类型 | 说明 |
|---|---|---|---|
| number | min/max/message | number | 数值范围 |
| array | min/max/message | checkbox / upload | 数组长度（"至少 N 个"用这个） |
| string | min/max/message | input | 字符串长度（生产只在 type=number 时校验 min/max，string 不生效，建议改用 len 或 custom） |
| pattern | pattern/message | 全部 | 正则字符串（不带斜杠） |
| len | len/message | 全部 | 定长（如身份证 18 位） |
| enum | enumStr(逗号分隔)/message | select/radio | 枚举校验 |
| custom | custom/message/depends | 全部 | 表达式 `value >= bm && value <= bl`，可附 depends 条件 |
| validator | validator(函数体)/message/depends | 全部 | JS 函数体 `(value, fm, extra) => return true`，最灵活 |

> 默认只显示 number/array 两种；勾选"高级"展开 string/pattern/len/enum/custom/validator。

#### custom 表达式

可访问：
- `value`：当前字段值
- `fm.字段ID`：其他字段值
- `extra.envFlag`：渲染端标识（'portal' / 'platform'）

#### validator 函数体

```js
// 配置项里写：
return value != null && value >= fm.bm_r42 && value <= fm.bl_r42;
```

#### depends 附加条件

表达式为**假**时跳过本规则。常用于"审核驳回时才校验"：

```
depends: fm.audit_status === 'reject'
```

### 4.7 联动 linkage（select / radio）

```
当选择 [ trigger ▾ ] 时执行：
  ┌─────────────────────────────────────────────┐
  │ [动作类型 ▾] [目标控件 ▾] [值(仅 setValue)] [×] │
  │ [动作类型 ▾] [目标控件 ▾]                  [×] │
  └─────────────────────────────────────────────┘
  [+ 添加动作]

  （若动作是 rowEnable/rowDisable/rowClear）
  「rowEnable」行绑定 rowTargets：
    [行号] [该行控件多选 ▾]                  [×]
    [行号] [该行控件多选 ▾]                  [×]
    [+ 添加行]
```

| 动作类型 | 字段 | 说明 |
|---|---|---|
| enable / disable / show / hide / clear / setValue | target | 单目标，target=cell id |
| rowEnable / rowDisable / rowClear | rowTargets[{row, cols[]}] | 多目标批量（cols 是该行 cell id 列表） |

> rowTargets.row 仅人工可读，引擎按 cols 寻址。删除行时引擎自动同步 row 号。

### 4.8 事件 events（可交互控件）

函数签名 `(value, fm, extra, rawEvent) => { 函数体 }`：

| 事件 | 触发时机 | 适用类型 | 典型用途 |
|---|---|---|---|
| onChange | 值变化后 | 全部可交互控件 | 跨字段重算 / 联动赋值 |
| onBlur | 失焦时 | input / textarea | 格式化（trim、大小写） |
| onInput | 输入中（每字符） | input / textarea | 高频，慎用重逻辑 |

**函数体 4 个参数**：

| 参数 | 类型 | 说明 |
|---|---|---|
| `value` | any | 当前字段新值（已写入 `fm[id]`） |
| `fm` | object | 整个表单 model，**可读可写**（`fm.total = 100` 触发响应式） |
| `extra` | `{cell, cellMap, envFlag}` | cell=配置对象，cellMap=id→cell 字典，envFlag='portal'/'platform' |
| `rawEvent` | iView 原参 | input-number/select/radio/checkbox=选中值；date={date: Date, label}；input on-blur=FocusEvent；input onInput=输入字符串 |

**写法示例**：
```js
// 跨字段重算
fm.total = (Number(fm.a) || 0) + (Number(fm.b) || 0)

// date-picker 拿 Date 对象
fm.date_str = rawEvent.date ? rawEvent.date.toISOString() : ''

// 按端分支
if (extra.envFlag === 'portal') fm.score = value * 0.5
else fm.score = value * 1.0
```

执行失败静默（`console.error`，不中断主流程）。

### 4.9 状态（5 选 1 单选）

| 状态 | 字段组合 | 行为 |
|---|---|---|
| 正常 | （无） | 默认 |
| 只读 | `disabled: true` | 控件禁用 |
| 门户端隐藏 | `portalHide: true` | 企业端不渲染 |
| 管理端隐藏 | `platformHide: true` | 管理端/专家端不渲染 |
| 双端隐藏 | `portalHide + platformHide` | 调试用，全场景不渲染 |

> 隐藏的 cell 不参与列宽计算；整列无可见 cell 时列 collapse；整行无可见 cell 时整行不渲染。

## 五、预览区（preview-renderer 子组件）

### 5.1 预览工具栏

| 控件 | 行为 |
|---|---|
| 自动预览 switch | 开启时编辑抽屉保存后自动刷新预览 |
| 视角 radio | 全部 / 门户端 / 管理端 切换（影响 portalHide/platformHide 是否生效、envFlag 注入） |
| 单元格 / X / Y 统计 chip | 显示当前规模 |
| 重复 ID / 缺 ID chip | 标红显示，hover 看详情 |
| 刷新预览 | 手动触发重建 |
| 下载 JSON | 直接下载（不打开弹窗） |

### 5.2 预览渲染行为

- **表头吸顶**：`<thead>` 内含 axis-row-top（行号列头）+ 前 `freezeRows` 行
- **行号列**：左侧 axis-row-head 显示行号
- **彩色徽章**：每个 cell 角落显示 id 徽章（双端隐藏灰色、门户隐藏红色、管理端隐藏橙色、可见绿色），点击打开编辑抽屉
- **空位 +**：未配置 cell 的位置显示 +，点击在该位置添加 cell
- **空位扩展**：通过"追加列/行"扩展出的虚拟空位（gridPad），允许在末尾追加新 cell

### 5.3 提交测试

预览区有"提交"按钮（在 preview-renderer 内部，触发 `handleSubmit`）：
- 跑全表校验
- 通过：`$emit('submit', result)`，控制台输出 config / fm / 过滤后 result
- 失败：`$emit('validate-fail', result)`，控制台 warn

## 六、应用到整列（批量操作）

cell-card 编辑抽屉里有"应用到整列"按钮，打开批量 modal：

```
┌──────────────────────────────────────────────┐
│ 复制字段（勾选要批量应用的字段）                 │
│ ☑ type  ☑ placeholder  ☑ precision           │
│ ☑ options  ☑ validation  ☑ accept  ☑ maxCount│
│ ☑ clampLines  ☐ content  ☐ value  ☐ width    │
├──────────────────────────────────────────────┤
│ 目标单元格（共 N 个）                          │
│ [全选] [排除表头] [仅未配置] [清空]            │

│ ☐ c9 (number) 未配置                         │
```

- **目标筛选**：4 种快捷（全选 / 排除表头 / 仅未配置 / 清空）
- 应用后入撤销栈

## 七、JSON 序列化（serializeJSON）

### 7.1 顶层结构

```json
{
  "title": "通则指标评分表",
  "freezeRows": 1,
  "cells": [
    { "id": "h1", "x": "1", "y": "1", "type": "label", "content": "一级指标" },
    { "id": "c8", "x": "4", "y": "2", "type": "number", "value": null,
      "validation": { "rules": [...] },
      "events": { "onChange": "..." } }
  ]
}
```

### 7.2 字段输出规则（_stripInternal + 空值清理）

- 永远输出：`id / x / y / type`、`title`、`freezeRows`（含 0）
- 非空才输出：`content / value / placeholder / options / calculationFormula / accept / maxCount / disabled / _hidden / portalHide / platformHide / clampLines / width / precision`
- 有内容才输出：`validation` / `linkage` / `events`

### 7.3 cell 字段输出顺序

`{ id, x, y, type, content?, value?, placeholder?, options?, calculationFormula?, accept?, maxCount?, disabled?, _hidden?, portalHide?, platformHide?, clampLines?, width?, precision?, validation?, linkage?, events? }`

### 7.4 events 输出

只输出 `onChange / onBlur / onInput` 三种里非空的，去掉前后空白：

```json
"events": {
  "onChange": "fm.total = Number(fm.a) + Number(fm.b)",
  "onBlur": "fm.indval_r3 = (fm.indval_r3||'').trim()"
}
```

## 八、JSON 导入（doImport）

### 8.1 入口

- 点"导入 JSON" → 弹窗粘贴 textarea + 选文件按钮
- 选 `.json` 文件后**直接应用**（无需再点导入）；粘贴路径仍需点导入

### 8.2 校验流程

1. `parseConfigText(text)` 解析（容错：尾逗号清理）
2. schema 校验：
   - 必须包含 `cells` 数组
   - 每个 cell 必须有 `id` / `x` / `y`（缺一报错）
   - `type` 必须在 11 种枚举内
   - ID 重复告警（允许导入，cellMap 后者覆盖前者）
3. 弹确认框（显示统计 / 警告 / 错误）
4. 应用配置 → 入撤销栈

## 九、与生产组件的对齐

配置页输出 → 生产 [dynamic-eval-table.vue](F:/0_workspace/3-省厅扶持库/project-support-library-platform/src/components/dynamic-eval-table/dynamic-eval-table.vue) 直接消费。关键对齐点：

| 字段 / 行为 | 配置页 | 生产 |
|---|---|---|
| `freezeRows` | 顶部 InputNumber | `<thead>` 吸顶前 N 行 |

## 十一、操作编辑详述（每个控件 → 函数 → 副作用）

> 本节穷举编辑抽屉中每个控件的"点击 → 触发函数 → 写入字段 → 通知主应用"链路。
> 所有写入走 `this.$set(this.cell, ...)` 保证响应式，通知走 `this.$emit('change')` 冒泡到主应用的 `schedulePreview`。

### 11.1 基础字段（cell-card methods）

| 控件 | 触发函数 | 写入字段 | $emit('change')? |
|---|---|---|---|
| ID 输入框 | cell-body `@change` 冒泡 | `cell.id`（v-model 双向） | 失焦时由 `@change` 冒泡 |
| 类型 i-Select | `onTypeChange` | 切到 options 类自动建 `cell.options=[]`；切到 formula 建 `cell.calculationFormula=''`；切离 input/textarea 清理 onBlur/onInput | 立即 |
| 列坐标 x / 行坐标 y | cell-body `@change` 冒泡 | `cell.x` / `cell.y` | 失焦时 |
| 初始值 value | cell-body `@change` 冒泡 | `cell.value` | 失焦时 |
| 显示内容 content | cell-body `@change` 冒泡 | `cell.content` | 失焦时 |
| 占位提示 placeholder | cell-body `@change` 冒泡 | `cell.placeholder` | 失焦时 |
| 数字精度 precision | cell-body `@change` 冒泡 | `cell.precision` | 失焦时 |
| 列宽权重 width | `setCellWidth(val)` | `$set(cell, 'width', val)` | **立即**（iView InputNumber 步进/Enter 不冒泡 DOM change，必须显式 $emit） |
| 溢出隐藏 clampLines | `setClampLines(val)` | `$set(cell, 'clampLines', val)` | **立即**（同 width） |
| 上传上限 maxCount | `setMaxCount(val)` | `$set(cell, 'maxCount', val)` | **立即**（同 width） |

> **为什么 width/maxCount/clampLines 要显式 $emit**：iView InputNumber 的 DOM `change` 事件只在**失焦**时派发，步进按钮、键盘 Enter 触发的 `on-change` 不会冒泡到外层 cell-body。若靠 `@change` 冒泡，预览不会刷新。

### 11.2 选项 options（select/radio/checkbox）

| 操作 | 触发函数 | 副作用 |
|---|---|---|
| 添加选项 | `addOption()` | `cell.options.push({label:'', value:'', score:null})` + $emit |
| 删除选项 | `removeOption(i)` | `cell.options.splice(i, 1)` + $emit |
| 改选项字段 | cell-body `@change` 冒泡 | v-model 直接写 `cell.options[i].label/value/score` |

### 11.3 公式 calculationFormula（formula 专用）

| 操作 | 触发函数 | 副作用 |
|---|---|---|
| 输入公式文本 | cell-body `@change` 冒泡 | v-model 写 `cell.calculationFormula` |
| 点快捷按钮 SUM/AVG/MIN/CALC_SCORE/CALC | `insertFormula(tpl)` | `cell.calculationFormula += tpl` + $emit |

### 11.4 必填设置（顶部 radio）

状态机：`'none' ↔ 'always' ↔ 'expr'`，约定 `validation.rules[0]` 是 required rule（kind='required'）。

| 操作 | 触发函数 | 副作用 |
|---|---|---|
| 切到"非必填" | `setRequiredState('none')` | 找到 rules 里 kind='required' 的删掉；不创建 |
| 切到"始终必填" | `setRequiredState('always')` | 找到或创建 required rule，`rule.required = true` |
| 切到"条件必填" | `setRequiredState('expr')` | 找到或创建 required rule，`rule.required = cell._requiredExprCache \|\| ''`（恢复上次表达式） |
| 输入表达式 | `setRequiredExpr(v)` | `rule.required = v`；非空时同步缓存到 `cell._requiredExprCache` |
| 错误提示 | `setRequiredMessage(v)` | `rule.message = v` |
| 触发时机 | `setRequiredTrigger(v)` | `rule.trigger = v` |

> **跨状态保留表达式**：从 `'expr'` 切到 `'always'` 再切回 `'expr'`，文本框会自动恢复上次输入的表达式（缓存在 `cell._requiredExprCache`，序列化时被忽略）。这避免误切状态丢失精心写的表达式。

### 11.5 其他校验规则（下方列表）

| 操作 | 触发函数 | 副作用 |
|---|---|---|
| 添加规则 | `addRule()` | 按 cell.type 推默认 kind（number→number/input→string/checkbox→array/其他→custom），push 新 rule + $emit |
| 删除规则 | `removeRule(i)` | 注意 i 是 otherRules（非 required）中的索引，需映射到 validationRules 全局索引后 splice + $emit |
| 切换 kind | `onRuleKindChange(r)` | 用对象替换（避免 Vue 2 新字段非响应式）：构造 `{kind, message, useDepends, ...kind 专属字段}` 后 `$set(rules, idx, nr)` + $emit |
| enum 输入逗号分隔 | `syncEnum(r)` | 拆分 `r.enumStr` 成数组写 `r.enum` + $emit |
| number/string/array min/max | cell-body `@change` 冒泡 | v-model 写 `r.min/r.max` |
| pattern 正则 | cell-body `@change` 冒泡 | v-model 写 `r.pattern` |
| len 定长 | cell-body `@change` 冒泡 | v-model 写 `r.len` |
| custom 表达式 | cell-body `@change` 冒泡 | v-model 写 `r.custom` |
| depends 附加条件 | cell-body `@change` 冒泡 | v-model 写 `r.depends` |
| validator 函数体 | cell-body `@change` 冒泡 | v-model 写 `r.validator` |

> **kind=depends / kind=required 兼容**：下拉里已无这两项，但代码保留分支以兼容老配置（防御性）。

### 11.6 联动 linkage（select/radio）

| 操作 | 触发函数 | 副作用 |
|---|---|---|
| 添加联动规则 | `addLinkage()` | `cell.linkage.push({trigger:'', actions:[]})` + $emit |
| 删除联动规则 | `removeLinkage(i)` | splice + $emit |
| 添加动作 | `addAction(lk)` | `lk.actions.push({kind:'enable', target:'', type:'enable'})` + $emit |
| 删除动作 | `removeAction(li, ai)` | splice + $emit |
| 切动作 kind | `onActionKindChange(a)` | 对象替换：`{kind, type, target, value?, rowTargets?}`，row* 类自动建 `rowTargets:[{row:1, cols:[]}]` + $emit |
| 添加行绑定 | `addRowTarget(a)` | `a.rowTargets.push({row:1, cols:[]})` + $emit |
| 删除行绑定 | `removeRowTarget(a, ri)` | splice + $emit |
| 行号 cols 多选 | cell-body `@change` 冒泡 | v-model 写 `rt.cols` |
| 触发值 trigger | cell-body `@change` 冒泡 | v-model 写 `lk.trigger` |
| 目标控件 target | cell-body `@change` 冒泡 | v-model 写 `a.target` |
| setValue 值 | cell-body `@change` 冒泡 | v-model 写 `a.value` |

### 11.7 事件 events（可交互控件）

| 操作 | 触发函数 | 副作用 |
|---|---|---|
| 输入 onChange / onBlur / onInput | `setEvent(eventType, body)` | 按需建 `cell.events={}`；非空 body 写入 `cell.events[eventType]`，空 body 用 `$delete` 移除 |

> **不立即 $emit('change')**：textarea 的 `@on-blur` 会冒泡到 cell-body 触发 `@change` → `schedulePreview`，避免每个字符都刷新预览。

### 11.8 状态（5 选 1 单选）

|---|---|---|
| 切状态 | `setVisibilityState(state)` | 同时写 `disabled` / `portalHide` / `platformHide` 三个字段 + $emit |

状态映射：
- `'normal'` → 三字段全 false
- `'disabled'` → disabled=true，hide 字段 false
- `'portalHide'` → portalHide=true，其他 false
- `'platformHide'` → platformHide=true，其他 false
- `'dualHide'` → portalHide=true, platformHide=true

## 十二、点击操作 → 渲染逻辑

> 本节按"用户在哪里点了什么"组织，对应"主应用/预览子组件"的响应链路。

### 12.1 顶部工具栏按钮

| 点击 | 函数 | 渲染影响 |
|---|---|---|
| + 添加单元格 | `addCellAndEdit()` → 打开模板 modal → 选模板后 `addFromTemplate(tpl)` | push 新 cell + 自动开抽屉 + schedulePreview（400ms 防抖） |
| 列表 | `listDrawerOpen = true` | 打开列表抽屉，渲染所有 cell-card |
| 导入 JSON | 弹导入 modal → `_applyPendingImport()` | _pushUndo + 整体替换 config + 重置 gridPad + forceRefreshPreview |
| 导出 JSON | 打开导出弹窗 | 不影响预览，仅展示 `serializeJSON(pretty)` |
| 复制 JSON | `serializeJSON(false)` → 剪贴板 | 不影响预览 |
| 清空 | `confirmClear()` | 二次确认 → config.cells=[] + gridPad={0,0} + schedulePreview |
| 载入示例 | `loadSample()` | config = makeSample() + schedulePreview |

### 12.2 行列操作按钮（顶部）

所有行列操作走相同模式：**`_pushUndo()` → 调 DTUtils → 同步 gridPad → `forceRefreshPreview()`**

| 点击 | 函数 | DTUtils 调用 | gridPad 同步 |
|---|---|---|---|
| 插入列 | `confirmInsertCol()` → `insertColumnAt(at, count)` | `insertColumns(cells, at, count)` | 若 cols>0，cols+=count |
| 插入行 | `confirmInsertRow()` → `insertRowAt(at, count)` | `insertRows(cells, at, count)` | 若 rows>0，rows+=count |
| 追加列 | `confirmAppendCol()` → `appendColumnsOp(count)` | 无（直接扩 gridPad） | cols = max(cols, maxX) + count |
| 追加行 | `confirmAppendRow()` → `appendRowsOp(count)` | 无 | rows = max(rows, maxY) + count |
| 删除列 | `confirmDeleteCol()` → `deleteColumn(col)` | `removeColumn(cells, col)` | cols = max(0, cols-1) |
| 删除行 | `confirmDeleteRow()` → `deleteRow(row)` | `removeRow(cells, row)` | rows = max(0, rows-1) |
| 列可见性 | `confirmColVis()` | 无（直接 $set cells 的 portalHide/platformHide） | 无 |

> **`forceRefreshPreview` vs `schedulePreview`**：行列操作用 forceRefreshPreview（立即刷新，无防抖），因为用户点 modal "确认"后期望立即看到结果；编辑抽屉用 schedulePreview（400ms 防抖）容忍连续输入。

### 12.3 撤销/重做

| 操作 | 函数 | 行为 |
|---|---|---|
| Ctrl+Z / ↶ 按钮 | `undo()` | redoStack.push(当前快照) → _restoreSnap(undoStack.pop()) → forceRefreshPreview |
| Ctrl+Y / Ctrl+Shift+Z / ↷ 按钮 | `redo()` | undoStack.push(当前快照) → _restoreSnap(redoStack.pop()) → forceRefreshPreview |

**入栈时机**：`_pushUndo()` 仅在这些操作前调用：
- 行列操作（insert/remove/append）
- 应用到整列（applyToColumn）
- 列可见性（confirmColVis）
- 导入 JSON（_applyPendingImport）
- **单 cell 编辑抽屉的"保存"按钮**（`commitEditDrawer`，2026-07-01 修复）

**不入栈的操作**：
- cell-card 内的字段修改（`@change` 只触发 schedulePreview；草稿未 commit 前用"取消"按钮丢弃）
- 顶部标题/freezeRows 输入
- gridPad 单独变化（仅作为 undo/redo 状态的一部分被快照保存）

> **设计权衡**：cell-card 字段改动太频繁（每次输入都入栈会爆栈），用抽屉的"取消"按钮 revert 草稿；点"保存"后才入栈，可 Ctrl+Z 撤销整个保存动作。

### 12.4 预览区点击
