# 动态表格渲染引擎 — 后端数据配置文档

> 本文档面向后端开发人员，说明如何生成、存储、下发动态表格的 JSON 数据配置。
> 后端只需按照本文档规定的 JSON Schema 生成数据，前端渲染引擎即可自动完成表格渲染、控件交互、联动、校验。

---

## 一、数据总结构

```json
{
  "title": "表格标题",
  "cells": [ 单元格对象1, 单元格对象2, ... ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | 否 | 表格标题，显示在表格上方居中位置 |
| `cells` | array | 是 | **扁平化**的单元格数组，引擎自动根据坐标计算合并与布局 |

> **核心思想**：后端无需关心表格的合并逻辑，只需为每个单元格指定坐标 `(x, y)` 和合并范围，引擎自动计算 `rowspan`/`colspan`。

---

## 二、单元格字段定义（Cell）

每个 cell 是一个 JSON 对象，完整字段如下：

```json
{
  "id": "c8",
  "x": "8",
  "y": "4",
  "type": "number",
  "content": "",
  "value": null,
  "placeholder": "输入标杆值",
  "options": null,
  "validation": { "rules": [...] },
  "linkage": [...],
  "calculationFormula": null,
  "disabled": false,
  "_hidden": false
}
```

### 字段详细说明

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | string | **是** | — | 全局唯一标识符。用于表单绑定、联动目标引用、公式引用。建议使用有意义的英文命名，如 `r2_benchmark` |
| `x` | string | **是** | — | 列坐标。单列 `"3"`，合并多列 `"3,5"` 表示第3~5列合并（闭区间）。**从1开始** |
| `y` | string | **是** | — | 行坐标。单行 `"2"`，合并多行 `"2,7"` 表示第2~7行合并（闭区间）。**从1开始** |
| `type` | string | **是** | — | 单元格类型，见【第三节 类型清单】 |
| `content` | string | 否 | `""` | 显示文本。支持 HTML 标签（如 `<b>加粗</b>`）和 `\n` 换行符。对 `label`/`text` 类型为显示内容；对 `select` 类型为 placeholder 提示 |
| `value` | any | 否 | `null` | 控件初始值/默认值。提交时前端会回传相同结构的 `{id, type, value}` |
| `placeholder` | string | 否 | `""` | 输入框占位提示文字（仅 `input`、`number` 类型有效） |
| `options` | array | 否 | `null` | 选项列表，仅 `select`/`radio`/`checkbox` 类型需要配置，见【第四节】 |
| `validation` | object | 否 | `null` | 校验规则配置，见【第五节】 |
| `linkage` | array | 否 | `null` | 联动规则数组，见【第六节】 |
| `calculationFormula` | string | 否 | `null` | 计算公式，仅 `formula` 类型使用，见【第七节】 |
| `disabled` | boolean | 否 | `false` | 初始是否禁用。`true` 时控件灰化不可操作，常用于联动受控控件 |
| `_hidden` | boolean | 否 | `false` | 初始是否隐藏。`true` 时不渲染可见内容，常用于联动备选项 |
| `accept` | string | 否 | `""` | (upload 专用) 限制上传文件格式，同时作为 `<input accept>` 属性。如 `".pdf,.doc,.docx"` |
| `maxSize` | number | 否 | — | (upload 专用) 最大文件大小，单位 MB。超过时拒绝上传并提示 |

---

## 三、单元格类型（type）

| type | 说明 | 渲染效果 | 需配置的字段 | value 类型 |
|------|------|----------|-------------|-----------|
| `label` | 表头标签 | `<th>` 加粗居中 | content | 不需要 value |
| `text` | 静态文本 | `<td>` 内文本，支持 HTML | content | 不需要 value |
| `input` | 文本输入框 | iView Input | placeholder, validation | string |
| `number` | 数字输入框 | iView InputNumber | placeholder, validation | number / null |
| `select` | 下拉选择 | iView Select + Option | options, validation, linkage | string |
| `radio` | 单选按钮组 | iView RadioGroup + Radio | options, validation, linkage | string |
| `checkbox` | 多选框组 | iView CheckboxGroup + Checkbox | options, validation | array |
| `date` | 日期选择器 | iView DatePicker | validation | string (日期字符串) |
| `upload` | 文件上传 | 文件选择 + 预览 | accept, maxSize, validation | string (JSON) |
| `formula` | 计算结果（只读） | 加粗蓝色文本 | calculationFormula | 不需要 value |

### 类型选择指南

```
需要用户输入？ ─── 否 ──→ 是表头？ ──→ label
                │              否 ──→ 是计算结果？ ──→ formula
                │                      否 ──→ text
                │
                └── 是 ──→ 是数字？ ──→ number
                         │  是日期？ ──→ date
                         │  是多选？ ──→ checkbox
                         │  是单选且选项少？ ──→ radio
                         │  是单选且选项多？ ──→ select
                         │  普通文本？ ──→ input
```

## 四、选项配置（options）

适用于 `select`、`radio`、`checkbox` 类型。

```json
"options": [
  { "label": "显示文字1", "value": "val1" },
  { "label": "显示文字2", "value": "val2" },
  { "label": "显示文字3", "value": "val3" }
]
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `label` | string | 是 | 选项显示文字。支持 HTML 标签 |
| `value` | string/number/boolean | 是 | 选中后的值，存入表单模型。**联动规则中的 trigger 需要匹配此值** |

### 配置要点

- `label` 是用户看到的文字，`value` 是程序使用的值，两者可以不同
- `value` 必须是字符串类型（用引号包裹），因为联动匹配时使用字符串比较
- 选项数量 ≤ 4 个建议用 `radio`，> 4 个建议用 `select`

## 五、校验规则配置（validation）

校验完全由数据驱动，前端不硬编码任何校验逻辑。

### 5.1 格式1：rules 数组（推荐，支持多规则）

```json
"validation": {
  "rules": [
    { "required": true, "message": "请输入标杆值" },
    { "type": "number", "min": 0, "max": 100, "message": "请输入0-100之间的值" }
  ]
}
```

### 5.2 格式2：简写（单规则快捷写法）

```json
"validation": {
  "required": true,
  "message": "此项为必填"
}
```

### 5.3 可用规则字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `required` | boolean | 是否必填 | `true` |
| `message` | string | 校验失败提示文字 | `"请输入标杆值"` |
| `trigger` | string | 触发时机：`"blur"`（失焦）/ `"change"`（值变化）。**不配置时自动推断**：number/select/radio 默认 change，其他默认 blur | `"blur"` |
| `type` | string | 值类型校验：`"string"`、`"number"`、`"array"`、`"email"`、`"url"` 等 | `"number"` |
| `min` | number | 最小值（type=number）或最小长度（type=string） | `0` |
| `max` | number | 最大值（type=number）或最大长度（type=string） | `100` |
| `len` | number | 精确长度 | `11` |
| `pattern` | string | 正则表达式（字符串格式） | `"^1[3-9]\\d{9}$"` |
| `enum` | array | 枚举值白名单 | `["A","B","C"]` |
| `whitespace` | boolean | 是否允许纯空格（配合 required） | `true` |
| `depends` | string | 条件表达式，**仅在表达式求值为 true 时才应用此规则**。表达式中可引用任意 cell id，引擎会自动替换为当前表单值。见【5.6 条件校验】 | `"bm_r42 != null"` |
| `custom` | string | 自定义校验表达式，表达式中 `value` 代表当前字段值，其他 cell id 替换为表单值。表达式求值为 `true` 校验通过，`false` 校验失败。见【5.7 自定义校验】 | `"value >= bm_r42 && value <= bl_r42"` |

### 5.4 自动推断行为

后端**不需要**配置 `trigger` 和 `type`，引擎会自动处理：

- `trigger`：`number`/`select`/`radio` 类型默认 `"change"`，`input`/`date` 默认 `"blur"`
- `type`：`number` 类型 + `required` 时自动补 `"number"`

### 5.5 常用配置示例

```json
// 必填
"validation": { "rules": [{ "required": true, "message": "此项为必填" }] }

// 必填 + 数值范围
"validation": { "rules": [
  { "required": true, "message": "请输入标杆值" },
  { "type": "number", "min": 0, "max": 100, "message": "请输入0-100之间的值" }
]}

// 仅范围校验（非必填，但填了就校验范围）
"validation": { "rules": [{ "type": "number", "min": 0, "message": "不能为负数" }] }

// 正则校验
"validation": { "rules": [{ "pattern": "^1[3-9]\\d{9}$", "message": "请输入正确的手机号" }] }
```

### 5.6 条件校验（depends）

当校验规则需要根据**其他单元格的值**动态生效时，使用 `depends` 字段。

#### 基本语法

```json
"validation": {
  "rules": [
    {
      "required": true,
      "message": "标杆值已填写，请上传证明材料",
      "depends": "bm_r42 != null"
    }
  ]
}
```

含义：只有当 cell `bm_r42` 的值不为 null 时，此 required 规则才生效。如果 `bm_r42` 为 null，规则被跳过（不会校验失败）。

#### 表达式语法

| 场景 | depends 值 | 说明 |
|------|-----------|------|
| 有值即生效 | `"bm_r42 != null"` | 标杆值非空时必填 |
| 等于某值 | `"c4 === 'has'"` | 选择"有适用标准"时必填 |
| 不等于某值 | `"c4 !== 'no'"` | 不是"无标准"时必填 |
| 多个条件 AND | `"bm_r42 != null && bl_r42 != null"` | 标杆值和基准值都填时必填 |
| 任一满足 OR | `"bm_r42 != null || indval_r42 != null"` | 任一有值时必填 |
| 数字比较 | `"bm_r42 > 10"` | 标杆值大于10时必填 |
| 三元表达式 | `"bm_r42 > 0 ? true : false"` | 复杂条件判断 |

#### 类型替换规则

引擎会将表达式中的 cell id 替换为当前表单值：

| fm 中的值类型 | 替换为 | 示例 |
|--------------|--------|------|
| `null` / `undefined` | `null` | `bm_r42` → `null` |
| number | 数字字面量 | `bm_r42` → `2` |
| string | 带引号的字符串 | `c4` → `'has'` |

#### 安全机制

- 引用不存在的 cell id → 表达式求值失败 → 规则跳过（不会误报错）
- 表达式语法错误 → 规则跳过
- 空表达式 → 规则跳过

#### 完整示例：上传文件依赖标杆值

```json
{
  "id": "evidence_upload_42",
  "x": "12", "y": "42",
  "type": "upload",
  "validation": {
    "rules": [
      {
        "required": true,
        "message": "标杆值已填写，请上传证明材料",
        "depends": "bm_r42 != null"
      }
    ]
  }
}
```

同行的标杆值单元格：
```json
{
  "id": "bm_r42",
  "x": "8", "y": "42",
  "type": "number",
  "value": 2,
  "placeholder": "输入标杆值"
}
```

效果：用户填写标杆值后，上传附件变为必填；未填标杆值时，上传不做必填校验。

### 5.7 自定义校验（custom）

当校验逻辑超出 `required`/`min`/`max`/`pattern` 的能力范围时，使用 `custom` 表达式。

**表达式规则**：
- `value` 代表当前字段的值
- 其他 cell id 替换为 `fm` 中的当前值
- 求值为 `true` → 校验通过
- 求值为 `false` → 校验失败，显示 `message`

#### 示例：指标值必须在标杆值和基准值之间

```json
{
  "id": "indval_r42",
  "type": "number",
  "validation": {
    "rules": [
      { "required": true, "message": "请输入指标值" },
      { "type": "number", "min": 0, "message": "指标值不能为负" },
      {
        "message": "指标值必须在标杆值和基准值之间",
        "depends": "bm_r42 != null && bl_r42 != null",
        "custom": "value >= bm_r42 && value <= bl_r42"
      }
    ]
  }
}
```

这里 `depends` + `custom` 配合使用：
1. `depends` 判断标杆值和基准值是否都已填写 → 未填则跳过此规则
2. `custom` 判断当前值是否在范围内 → 不在范围则报错

#### custom 独用（无 depends）

```json
{
  "message": "指标值必须大于标杆值",
  "custom": "value > bm_r42"
}
```

无 `depends` 时规则始终生效。

### 5.8 upload 文件校验

upload 类型支持三个层次的校验：

| 层次 | 配置 | 校验时机 |
|------|------|---------|
| 是否必填 | `required` + `depends` | 表单提交时 |
| 文件格式 | `accept` 字段 | 上传时即时校验 |
| 文件大小 | `maxSize` 字段 | 上传时即时校验 |

```json
{
  "id": "evidence_upload_42",
  "type": "upload",
  "accept": ".pdf,.doc,.docx,.png,.jpg,.jpeg",
  "maxSize": 10,
  "validation": {
    "rules": [
      {
        "required": true,
        "message": "标杆值已填写，请上传证明材料",
        "depends": "bm_r42 != null"
      }
    ]
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `accept` | string | 允许的文件扩展名列表，逗号分隔。同时作为 HTML `<input accept>` 属性。不配置则不限制格式 |
| `maxSize` | number | 最大文件大小，单位 MB。不配置则不限制大小 |

## 六、联动配置（linkage）

联动实现控件间的交互逻辑：选择 A 选项 → 自动启用/禁用/显示/隐藏/清空其他控件。

### 6.1 基本结构

```json
"linkage": [
  {
    "trigger": "触发值（必须匹配 options 中的 value）",
    "actions": [
      { "target": "目标cell的id", "type": "动作类型" },
      { "type": "行绑定动作类型", "rowTargets": [...] }
    ]
  }
]
```

### 6.2 单目标动作

| type | 说明 | 示例场景 |
|------|------|----------|
| `show` | 显示目标单元格（取消隐藏） | 选择"有"→ 显示输入框 |
| `hide` | 隐藏目标单元格 | 选择"无"→ 隐藏输入框 |
| `enable` | 启用目标控件（取消禁用） | 选择"有标准"→ 启用数值输入 |
| `disable` | 禁用目标控件（灰化） | 选择"无标准"→ 禁用数值输入 |
| `clear` | 清空目标控件的值 | 切换选项时清空旧值 |
| `setValue` | 设置目标控件的值 | 自动填入固定值 |

> **特殊目标**：`target: "submit_btn"` 可控制提交按钮的禁用/启用，用于"一票否决"场景。

### 6.3 多目标行绑定动作

当一个选项需要同时操作多个控件时，使用 `rowTargets` 避免重复配置：

| type | 说明 |
|------|------|
| `rowEnable` | 批量启用 `cols` 中列出的所有控件 |
| `rowDisable` | 批量禁用 `cols` 中列出的所有控件 |
| `rowClear` | 批量清空 `cols` 中列出的所有控件值 |

```json
{
  "type": "rowEnable",
  "rowTargets": [
    {
      "row": 2,
      "cols": ["r2_benchmark", "r2_baseline", "r2_weight", "r2_indvalue"]
    }
  ]
}
```

> **关于 `row` 字段**：`rowTargets` 中的 `row` 字段仅为**人工可读的注释**，引擎不使用它。实际操作目标完全由 `cols` 数组中的 **cell id** 决定。引擎通过 id 在 `disabledState`/`hiddenState`/`fm` 中查找对应控件，与控件在哪一行哪一列无关。
>
> 换句话说：联动的寻址机制是**纯 id 寻址**，与坐标无关。`target: "c8"` 操作的就是 id 为 `"c8"` 的那个控件，不管它在表格的什么位置。

### 6.4 备选项切换（同位置联动）

两个 cell 的 `x`、`y` 完全相同时，后定义的作为备选项。备选项用 `_hidden:true` 初始隐藏。

```json
// 默认显示的输入框
{ "id": "d_energy", "x": "4", "y": "5", "type": "number", "placeholder": "综合能耗" }

// 备选项：无适用标准时显示 "/"
{ "id": "d_none", "x": "4", "y": "5", "type": "text", "content": "/", "_hidden": true }
```

联动时通过 `show`/`hide` 切换两者。

### 6.5 联动配置示例

#### 示例1：select 切换启用/禁用

```json
{
  "id": "c4", "x": "4", "y": "4", "type": "select",
  "options": [
    {"label":"有适用标准","value":"has"},
    {"label":"无适用标准","value":"no"}
  ],
  "linkage": [
    {
      "trigger": "has",
      "actions": [
        {"target":"c8","type":"enable"},
        {"target":"c9","type":"enable"},
        {"target":"d_energy","type":"show"},
        {"target":"d_none","type":"hide"}
      ]
    },
    {
      "trigger": "no",
      "actions": [
        {"target":"c8","type":"disable"},
        {"target":"c8","type":"clear"},
        {"target":"c9","type":"disable"},
        {"target":"c9","type":"clear"},
        {"target":"d_energy","type":"hide"},
        {"target":"d_energy","type":"clear"},
        {"target":"d_none","type":"show"}
      ]
    }
  ]
}
```

#### 示例2：radio 行绑定联动

```json
{
  "id": "sec_ind", "x": "3", "y": "2,3", "type": "radio",
  "options": [
    {"label":"选项1说明","value":"opt1"},
    {"label":"选项2说明","value":"opt2"}
  ],
  "linkage": [
    {
      "trigger": "opt1",
      "actions": [
        {"type":"rowEnable",  "rowTargets":[{"row":2, "cols":["r2_benchmark","r2_baseline"]}]},
        {"type":"rowDisable", "rowTargets":[{"row":3, "cols":["r3_benchmark","r3_baseline"]}]},
        {"type":"rowClear",   "rowTargets":[{"row":3, "cols":["r3_benchmark","r3_baseline"]}]}
      ]
    },
    {
      "trigger": "opt2",
      "actions": [
        {"type":"rowEnable",  "rowTargets":[{"row":3, "cols":["r3_benchmark","r3_baseline"]}]},
        {"type":"rowDisable", "rowTargets":[{"row":2, "cols":["r2_benchmark","r2_baseline"]}]},
        {"type":"rowClear",   "rowTargets":[{"row":2, "cols":["r2_benchmark","r2_baseline"]}]}
      ]
    }
  ]
}
```

## 七、公式配置（calculationFormula）

仅 `formula` 类型使用。

| 公式 | 语法 | 说明 |
|------|------|------|
| 求和 | `SUM(id1,id2,...)` | 引用 cell id 的值求和 |
| 平均值 | `AVG(id1,id2,...)` | 引用 cell id 的值求平均 |
| 最小值 | `MIN(id1,id2,...)` | 引用 cell id 的值取最小 |
| 全量求和 | `CALC_SCORE` | 所有 number 类型控件值自动求和 |
| 自定义计算 | `CALC(表达式)` | 标准 JS 表达式，cell id 自动替换为数值，可使用 Math 对象 |
| 四则运算 | `c6 * e5 / 100` | cell id 替换为数值后计算表达式 |

### CALC(表达式) 支持的特性

- 四则运算：`CALC((c8 - c9) / c10 * 100)`
- Math 函数：`CALC(Math.max(0, c8 - c9))`
- 三元表达式：`CALC(c8 > c9 ? c8 : c9)`
- 四舍五入：`CALC(Math.round(c8 * 100) / 100)`
- 范围裁剪：`CALC(c8 <= 0 ? 0 : c8 >= 100 ? 100 : c8)`

### CALC(表达式) 规则

- 表达式内部是标准 JavaScript 语法
- cell id 会被自动替换为对应数值（text 类型取 content 的数值，其他类型取 fm[id]）
- Math 对象的所有方法可用（Math.max, Math.min, Math.round, Math.floor, Math.ceil, Math.abs, Math.pow 等）
- 表达式中不能嵌套 SUM/AVG 等引擎函数（它们不是 JS 原生函数），只能用标准 JS 语法
- 计算出错时显示 "ERR"

```json
{ "id": "g3", "x": "6", "y": "8", "type": "formula", "calculationFormula": "SUM(c6,e5,f5)" }
{ "id": "g8", "x": "11", "y": "8", "type": "formula", "calculationFormula": "CALC_SCORE" }
{ "id": "g9", "x": "7", "y": "5", "type": "formula", "calculationFormula": "CALC((c8 - c9) / c10 * 100)" }
{ "id": "g10", "x": "8", "y": "5", "type": "formula", "calculationFormula": "CALC(Math.max(0, c8 - c9))" }
```

```json
       "options":[{"label":"有适用标准","value":"has"},{"label":"无适用标准","value":"no"}],

```

### 8.2 提交数据（用户填写后）

**接口**：`POST /api/table/submit`

**请求体**（前端生成）：

```json
{
  "tableId": "xxx",
  "data": {
    "title": "通则指标评分表",
    "cells": [
      {"id": "sec_ind", "type": "radio", "value": "opt1"},
      {"id": "r2_benchmark", "type": "number", "value": 120},
      {"id": "r2_baseline", "type": "number", "value": 80},
      {"id": "c4", "type": "select", "value": "has"},
      {"id": "c8", "type": "number", "value": 150},
      {"id": "c9", "type": "number", "value": 90}
    ]
  }
}
```

> 前端只提交 `{id, type, value}`，不包含 x、y、options 等配置信息。

### 8.3 回显数据（编辑已填表格）

**接口**：`GET /api/table/data/{recordId}`

**响应**：

```json
{
  "code": 0,
  "data": {
    "tableId": "xxx",
    "cells": [
      {"id": "sec_ind", "value": "opt1"},
      {"id": "r2_benchmark", "value": 120},
      {"id": "c4", "value": "has"},
      {"id": "c8", "value": 150}
    ]
  }
}
```

前端调用 `app.loadData(response.data)` 即可回显。

> 回显数据只需 `id` + `value`，不需要 `type`（前端已从配置中知道类型）。

## 九、数据存储建议

### 9.1 表配置表（table_config）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | varchar | 表格唯一标识 |
| title | varchar | 表格标题 |
| cells_json | text/json | 完整 cells JSON 数组 |
| version | int | 配置版本号 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

### 9.2 用户填写数据表（table_data）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | varchar | 记录唯一标识 |
| table_id | varchar | 关联 table_config.id |
| user_id | varchar | 填写人 |
| cells_data | text/json | `{cells: [{id, value}, ...]}` |
| status | int | 0=草稿, 1=已提交 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

### 9.3 设计原则

- **配置与数据分离**：表结构配置（cells_json）与用户填写数据（cells_data）分开存储
- **配置版本化**：修改配置不影响已提交的历史数据
- **cells_json 存完整配置**：包含所有字段的完整 cells 数组，前端拿到即可渲染
- **cells_data 只存 id+value**：最小化存储，回显时与配置合并
